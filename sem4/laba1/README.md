# Кадастровая система управления недвижимостью

## Описание проекта

Данный проект представляет собой программную реализацию кадастровой системы для учета объектов недвижимости, их владельцев и правоустанавливающих документов. Система автоматизирует процессы кадастрового и технического учета, генерации кадастровых номеров, а также предоставляет функционал для работы с выписками из реестра.

Проект разработан на языке Python с использованием объектно-ориентированного подхода и принципов предметно-ориентированного проектирования (Domain-Driven Design). Данные сохраняются в формате JSON, что обеспечивает простоту развертывания и не требует наличия отдельной базы данных.

## Основные возможности

1. **Управление объектами недвижимости**:
   - Создание и учет земельных участков
   - Создание и учет зданий
   - Хранение характеристик объектов (площадь, адрес, статус и др.)

2. **Управление владельцами**:
   - Регистрация физических лиц как правообладателей
   - Хранение контактных данных и идентификаторов (ИНН/паспорт)

3. **Регистрация прав**:
   - Оформление правоустанавливающих документов
   - Учет долей в праве собственности
   - Контроль суммы долей (не более 1)

4. **Кадастровый учет**:
   - Автоматическая генерация кадастровых номеров
   - Проверка уникальности номеров
   - Присвоение статусов кадастровым номерам

5. **Технический учет**:
   - Проведение технического обследования объектов
   - Генерация номеров технических паспортов
   - Фиксация технических характеристик

6. **Документооборот**:
   - Формирование выписок из ЕГРН
   - Проверка прав доступа к документам
   - Сохранение истории выписок

## Архитектура системы

Система построена по многослойной архитектуре с четким разделением ответственности:

### 1. Модели предметной области (Domain Models)
models/
- base_model.py # Абстрактный базовый класс с UUID
- dokument.py # Абстрактный класс документа
- kadastrovyj_nomer.py # Кадастровый номер
- objekt_nedvizhimosti.py # Базовый класс объекта недвижимости
- pravo_ustanavlivayushchij_dokument.py # Правоустанавливающий документ
- vladeshec.py # Владелец
- zdanie.py # Здание (наследует objekt_nedvizhimosti)
- zemelnyj_uchastok.py # Земельный участок (наследует objekt_nedvizhimosti)


**Ключевые особенности**:
- Все модели наследуются от `base_model`, который предоставляет уникальный идентификатор (UUID)
- Использование композиции: правоустанавливающий документ связывает объект и владельцев
- Инкапсуляция бизнес-правил в сеттерах (проверка долей, статусов и т.д.)

### 2. Репозитории (Repositories)

Репозитории отвечают за хранение и загрузку данных:
services/
- base_repository.py # Базовый репозиторий с JSON-сериализацией
- cadastral_number_repository.py # Репозиторий кадастровых номеров
- object_repository.py # Репозиторий объектов недвижимости
- owner_repository.py # Репозиторий владельцев
- right_document_repository.py # Репозиторий правоустанавливающих документов


**Особенности реализации**:
- Каждый репозиторий работает с отдельным JSON-файлом
- Реализованы методы `_to_dict` и `_from_dict` для сериализации/десериализации
- При загрузке данные преобразуются обратно в объекты моделей
- Репозитории автоматически создают директории при сохранении

### 3. Сервисы (Services)

Сервисы реализуют бизнес-операции и используют репозитории:
services/
- dokument_menedzher.py # Сервис документооборота
- kadastrovoe_agentstvo.py # Фасад - главный контроллер
- kadastrovyj_uchet.py # Сервис кадастрового учета
- number_generator.py # Генератор кадастровых номеров
- tekhnicheskiy_uchet.py # Сервис технического учета


**Ключевые сервисы**:

- **kadastrovoe_agentstvo** - фасад, объединяющий всю функциональность и предоставляющий единый интерфейс для пользовательского интерфейса
- **number_generator** - генерирует уникальные кадастровые номера с проверкой дубликатов
- **kadastrovyj_uchet** - выполняет постановку на кадастровый учет
- **tekhnicheskiy_uchet** - проводит техническое обследование и формирует техпаспорта
- **dokument_menedzher** - управляет выписками и проверяет права доступа

### 4. Исключения (Exceptions)

Иерархия исключений предметной области:
exceptions/
- exceptions.py


- **CadastralError** - базовое исключение
- **ValidationError** - ошибка валидации данных
- **NotFoundError** - сущность не найдена
- **DuplicateError** - дублирование данных
- **DocumentVerificationError** - ошибка проверки документа
- **PermissionDeniedError** - недостаточно прав

### 5. Пользовательский интерфейс

Консольное меню для взаимодействия с системой:
main.py (содержит класс CadastralMenu)


**Функции меню**:
- Интерактивный ввод с валидацией
- Отображение списков объектов и владельцев
- Обработка исключений и дружественные сообщения об ошибках
- Форматирование вывода (даты, статусы)

### 6. Константы и настройки
constants.py

Содержит:
- Параметры генерации кадастровых номеров
- Допустимые статусы объектов и номеров
- Градации износа и состояния
- Конфигурация JSON-сериализации
