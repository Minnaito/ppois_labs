# Лабораторная работа №1  
**Вариант: Система управления товарами (складской учёт)**

## Предметная область  
Управление складскими запасами товаров. Система обеспечивает ведение реестра товаров с информацией о производителе, количестве на складе, адресе хранения. Реализован полный цикл CRUD-операций с расширенным поиском и удалением по составным критериям, а также импорт/экспорт данных в XML.

## Важные последствия  
- Корректность и уникальность УНП производителя (9 цифр)  
- Некорректное количество на складе (неотрицательное целое)  
- Валидация всех входных полей  
- Сохранение данных в XML-файл с возможностью восстановления  
- Логика поиска/удаления: группировка условий с использованием **И** между группами и **ИЛИ** внутри группы

## Операции предметной области  
1. Добавление нового товара (с валидацией)  
2. Просмотр всех товаров с постраничным выводом  
3. Поиск товаров по комбинированным критериям (с пагинацией результатов)  
4. Удаление товаров по комбинированным критериям (с подтверждением)  
5. Экспорт всех данных в XML-файл (DOM)  
6. Импорт данных из XML-файла (SAX)  
7. Навигация по страницам (первая, последняя, следующая, предыдущая, переход к странице)  
8. Настройка количества записей на странице (5, 10, 20, 50, 100)

## Программа «Система управления товарами»  
Представляет собой модель складского учёта с графическим интерфейсом пользователя (GUI). Позволяет удобно управлять товарами, производить поиск и удаление по гибким правилам, а также обмениваться данными через XML.

Проект реализован на языке **Python** с использованием **Tkinter** (графический интерфейс) и архитектурного паттерна **MVC**.

## Реализация операций в коде  

| № | Операция | Реализация в коде |
|---|----------|-------------------|
| 1 | Добавление товара | `AddProductDialog`, метод `add_product` контроллера, `ProductList.add_product` |
| 2 | Просмотр всех товаров | `MainWindow.update_table`, `ProductList.get_page` с пагинацией |
| 3 | Поиск товаров | `SearchDialog` с группировкой условий (И/ИЛИ), `ProductList.search` |
| 4 | Удаление товаров | `DeleteDialog` с аналогичной логикой, `ProductList.delete_by_criteria` |
| 5 | Экспорт в XML | `XMLWriter.write_to_file` (DOM), метод `save_to_file` контроллера |
| 6 | Импорт из XML | `XMLReader.read_from_file` (SAX), метод `load_from_file` контроллера |
| 7 | Пагинация | Классы `MainWindow` и `SearchDialog`, методы `first_page`, `previous_page`, `next_page`, `last_page`, `go_to_page` |

## Графический интерфейс  
При запуске приложения открывается главное окно с таблицей товаров, панелью инструментов и меню.

### Главное окно
- Таблица со столбцами: Название товара, Название производителя, УНП производителя, Количество на складе, Адрес склада  
- Панель пагинации (выбор числа записей, кнопки навигации, поле ввода номера страницы)  
- Меню: **Файл** (Сохранить, Загрузить, Выход), **Правка** (Добавить, Поиск, Удалить), **Справка** (О программе)  
- Панель инструментов с кнопками для основных действий

### Диалоговые окна
- **Добавление товара** – поля: название товара, название производителя, УНП (9 цифр), количество на складе, адрес склада. Автоматическая валидация.  
- **Поиск товаров** – разделы: «Поиск по товару» (название, количество), «Поиск по производителю» (название, УНП), «Поиск по адресу» (адрес). Внутри каждой группы – логическое **ИЛИ**, между группами – **И**. Результаты выводятся в таблице с собственной пагинацией.  
- **Удаление товаров** – аналогичная структура выбора условий, подтверждение действия, отображение количества удалённых записей.

## Ключевые классы и их функции  

### MainController  
Главный контроллер (связующее звено между моделью и представлением).  
**Функции:**  
- Обработка действий пользователя (добавление, поиск, удаление, пагинация, импорт/экспорт)  
- Управление моделью `ProductList`  
- Обновление представления `MainWindow` через паттерн «Наблюдатель»  

### ProductList (модель)  
Управляет коллекцией товаров и логикой пагинации.  
**Функции:**  
- Добавление, удаление, очистка товаров  
- Поиск по критериям (частичное совпадение строк, точное совпадение УНП и количества)  
- Удаление по составным критериям (с проверкой каждого товара)  
- Пагинация: `get_page`, свойства `total_pages`, `current_page`, `rows_per_page`  
- Уведомление наблюдателей (View) об изменениях  

### Product  
Модель данных товара (dataclass).  
**Функции:**  
- Поля: `product_name`, `manufacturer_name`, `manufacturer_unp`, `stock_quantity`, `warehouse_address`  
- Метод `validate()` для проверки корректности  
- Преобразование в словарь и из словаря (`to_dict`, `from_dict`)  

### MainWindow (представление)  
Главное окно приложения.  
**Функции:**  
- Отображение таблицы товаров  
- Панель пагинации и элементы управления  
- Главное меню и панель инструментов  
- Методы обновления данных и отображения сообщений  

### AddProductDialog / SearchDialog / DeleteDialog  
Диалоговые окна для ввода данных.  
**Функции:**  
- Валидация ввода (непустые строки, формат УНП, целое число)  
- Построение критериев поиска/удаления с правилами И/ИЛИ  
- Отображение результатов поиска с собственной пагинацией (SearchDialog)  
- Подтверждение удаления с показом количества записей  

### XMLWriter / XMLReader / ProductHandler  
Работа с XML.  
**Функции:**  
- `XMLWriter.write_to_file` – экспорт всех товаров в XML с использованием DOM (`xml.dom.minidom`)  
- `XMLReader.read_from_file` – импорт из XML с использованием SAX (`xml.sax`)  
- `ProductHandler` – обработчик SAX, парсит XML и создаёт объекты `Product`  

### FileController  
Контроллер файловых операций (диалоги выбора файла, вызовы XMLWriter/XMLReader).  

### Конфигурация и константы  
- `config.py` – размеры окна, имена столбцов, параметры пагинации  
- `utils/constants.py` – перечисления сообщений для пользователя и сообщений валидации  

## Пагинация  
Пагинация реализована в двух местах:  
- **Главное окно** – все товары разбиты на страницы.  
- **Окно поиска** – результаты поиска также постранично отображаются.  

**Элементы управления:**  
- Кнопки «<<» (первая), «<» (предыдущая), «>» (следующая), «>>» (последняя)  
- Поле ввода номера страницы (переход по Enter)  
- Выбор количества строк на странице (5/10/20/50/100)  
- Отображение текущей страницы, общего числа страниц и общего количества записей  

## Логика поиска и удаления  

Правила формирования условий:  

| Группа | Условия внутри группы (логическое **ИЛИ**) |
|--------|----------------------------------------------|
| Группа 1 (товар) | Название товара содержит строку **ИЛИ** количество на складе равно значению |
| Группа 2 (производитель) | Название производителя содержит строку **ИЛИ** УНП производителя равно значению |
| Группа 3 (адрес) | Адрес склада содержит строку |

Между группами действует логическое **И**.

Пример:  
`(название товара содержит "мыло" ИЛИ количество = 0) И (название производителя содержит "Нэфис") И (адрес содержит "Казань")`

## Экспорт и импорт XML  

### Экспорт (DOM)  
- Вызывается через меню **Файл → Сохранить** или кнопку на панели инструментов  
- Выбор файла через стандартный диалог сохранения  
- Генерируется XML-файл с корневым элементом `<products>` и набором элементов `<product>`  
- Используется `xml.dom.minidom` для форматированного вывода  

### Импорт (SAX)  
- Вызывается через меню **Файл → Загрузить** или кнопку на панели инструментов  
- Выбор файла через стандартный диалог открытия  
- Парсинг XML с помощью `xml.sax` (энергоэффективно для больших файлов)  
- Добавление импортированных записей в существующий список (без удаления старых)  

## Тестирование программы  

В предоставленном коде отсутствуют автоматические тесты. Однако ключевые компоненты могут быть проверены следующим образом:

1. **Ручное функциональное тестирование** через GUI:
   - Добавление товаров с корректными и некорректными данными (проверка валидации)
   - Поиск с разными комбинациями условий
   - Удаление записей (проверка подсчёта удалённых)
   - Экспорт/импорт XML (сравнение данных до и после)
   - Пагинация (переключение страниц, смена количества записей)

2. **Блочное тестирование** (может быть реализовано с использованием `unittest`):
   - Модель `Product`: валидация полей, преобразование в словарь и обратно
   - Модель `ProductList`: добавление, поиск, удаление по критериям, пагинация
   - XML-парсеры: чтение/запись корректного и ошибочного XML

На момент сдачи лабораторной работы все заявленные функции работают корректно. Система готова к использованию.
